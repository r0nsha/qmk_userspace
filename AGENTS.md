# QMK Userspace — holykeebs / r0nsha

## One-line identity

Corne (crkbd/rev1) split keyboard userspace focused on multi-pointing-device support (trackball + trackpoint + touchpad combos). Shared code in `users/holykeebs/`, keymap in `keyboards/crkbd/rev1/keymaps/r0nsha/`.

## Build & flash

```bash
qmk compile -kb crkbd/rev1 -km r0nsha        # single compile
qmk userspace-compile                         # all targets (declared in qmk.json)
make crkbd/rev1:r0nsha                        # make passthrough
just flash                                    # stock OLED
just flash_trackpoint                         # trackpoint pointing device
```

`Makefile` forwards every target to `qmk_firmware` — any make invocation works.

## Dev setup

```bash
just install       # pip install qmk → qmk setup → compile with bear
just clone_firmware # clone holykeebs' qmk fork to ~/dev/qmk_firmware
just bear           # clean compile with bear for clangd
```

Uses `.devcontainer/` (QMK CLI image). Clangd needs `compile_commands.json` (generated by `just bear` or `just install`).

## Project structure

| Path                                   | What                                                                                                                                                                             |
| -------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `users/holykeebs/`                     | Shared pointing-device logic, EEPROM config, OLED, RPC state sync, pimoroni/trackpoint drivers                                                                                   |
| `users/holykeebs/rules.mk`             | ~600-line pointing device config matrix — conditionally selects drivers (`pimoroni_trackball`, `cirque_pinnacle_i2c`, `azoteq_iqs5xx`, `ps2`) based on `POINTING_DEVICE` env var |
| `keyboards/crkbd/rev1/keymaps/r0nsha/` | 8-layer keymap with OLED, encoder, RGB matrix, tap-dance                                                                                                                         |
| `qmk.json`                             | Build target: `crkbd/rev1:r0nsha`                                                                                                                                                |
| `Justfile`                             | Flash, setup, bear commands                                                                                                                                                      |

## Key build-time vars (set via `-e` or `#define`)

- `POINTING_DEVICE` — selects driver (`trackpoint`, `pimoroni`, `cirque`, `tps43`, `tps65`, etc.)
- `POINTING_DEVICE_POSITION` = left|right
- `OLED` = stock|yes (stock = standard OLED, yes = custom holykeebs OLED with logo)
- `USER_NAME` = holykeebs
- `HK_MASTER_LEFT` / `HK_MASTER_RIGHT` — overrides default MASTER side in `users/holykeebs/config.h`

## Keymap features (rules.mk flags)

`TAP_DANCE_ENABLE`, `CAPS_WORD_ENABLE`, `KEYCODE_STRING_ENABLE`, `MOUSEKEY_ENABLE`. VIA disabled.

## Custom keycodes (`users/holykeebs/holykeebs.h`)

| Code                            | Short alias   | Function                          |
| ------------------------------- | ------------- | --------------------------------- |
| `HK_SAVE_SETTINGS`              | `HK_SAVE`     | Save EEPROM config                |
| `HK_RESET_SETTINGS`             | `HK_RESET`    | Reset EEPROM config to defaults   |
| `HK_DUMP_SETTINGS`              | `HK_DUMP`     | Debug dump config to console      |
| `HK_SNIPING_MODE`               | `HK_SNIPING`  | Toggle sniping mode (slow cursor) |
| `HK_DRAGSCROLL_MODE`            | `HK_D_SCROLL` | Toggle drag-scroll                |
| `HK_CYCLE_SCROLL_LOCK`          | `HK_C_SCRL`   | Cycle scroll lock axis            |
| `HK_INVERT_SCROLL_DIR`          | `HK_I_SCRL`   | Invert scroll direction           |
| `HK_POINTER_SET_DEFAULT_SCALER` | —             | Set default cursor speed          |
| `HK_POINTER_SET_SNIPING_SCALER` | —             | Set sniping cursor speed          |
| `HK_POINTER_SET_SCROLL_BUFFER`  | —             | Set scroll accumulation buffer    |

## EEPROM config

`users/holykeebs/eeprom_config.h` — 64-byte persisted state via `EECONFIG_USER_DATA_SIZE`. Stores cursor modes, drag-scroll state, multipliers, scroll buffer per-side (main + peripheral). Versioned with checksum.

## Split sync

RPC-based state sync between halves via `users/holykeebs/rpc.c`. Uses QMK transaction API. Both main and peripheral pointer states synced.

## CI

`.github/workflows/build_binaries.yaml` — builds on every push via `qmk/.github` reusable workflows. Publishes to GitHub Releases.

## VCS

Uses both **git** and **Jujutsu (jj)** (`.jj/` at root). Either works for commits.

## Code style

Google style (clang-format), 1000-char column limit, 4-space indent, pointers right-aligned. `.mk` files use tabs (editorconfig).

## Common pitfalls

- `users/holykeebs/rules.mk` expects `POINTING_DEVICE` to be set — compiling without it may fail or produce wrong config
- `clangd` needs `compile_commands.json` — run `just bear` after any config change
- `qmk setup` (in `just install`) uses holykeebs' fork branch `hk-master`, not upstream QMK master
- EEPROM config format changes must keep backward compat or increment version field in `hk_eeprom_config_t`
